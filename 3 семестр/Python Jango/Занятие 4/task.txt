ПроектНазвание проекта: Шаблон MVC, контроллеры и маршруты.Цель работы: проверка знаний на практике.
Теория:
1.Что такое шаблон проектирования MVC?
2.Чем реализация шаблона MVC отличается в Django?
3. Что такое контроллеры?
Условие задания:
1. Создать контроллер-функцию для дополнительного приложения, созданного в прошлом домашнем задании. Вывести на страницу один-два html тэга.
2.В папке данного приложения создать файл urls.py для его маршрутов.
3.Подключить маршруты приложения к общим маршрутам

Ход выполнения:
1)В файле views.py второго приложения, созданного на прошлом уроке, создать контроллер-функцию с произвольным именем.
2)При помощи конструкции return HTTPResponse сделать так, чтобы в ответ за запрос контроллер возвращал какой-нибудь HTML-тэг с текстом.
3)В папке приложения создать файл urls.py для прописывания маршрутов
4)В файле общих маршрутов, т. е. в файле urls.py в папке News, прописать подключение маршрутов нового приложения в формате path(..., include(...))

Список терминов:Шаблонпроектирования—принцип построения приложения.MVC—шаблон проектирования «Модель-Представление-Контроллер». В фреймворке Django интерпретируется как MVT -«Модель-Контроллер-Шаблон».Контроллер—функция или класс, управляющая определенной функцией приложения. Позволяет связать представление(шаблон) и модель вместе.
